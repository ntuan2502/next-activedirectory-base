# Security Policy

We take security seriously and want to keep our project safe for everyone. Thank you for helping us maintain a secure application.

---

## 🛡️ Supported Versions

We actively monitor and patch vulnerabilities in the following versions of **Active Directory Sync**:

| Version | Supported |
| --- | --- |
| `<current_version>` / `main` branch | ✅ Supported |
| All older versions | ❌ Unsupported |

Please always ensure you are running the latest code from the `main` branch to receive active security updates.

---

## 🔒 Reporting a Vulnerability

If you discover a security vulnerability in this project, **please do not report it publicly via GitHub issues or discussions**. Instead, report it privately to the project maintainers.

### Reporting Procedure

1. **Email Your Findings**
   Send an email details to: **[admin@tun.io.vn](mailto:admin@tun.io.vn)**

2. **What to Include**
   To help us quickly understand and address the issue, please include:
   - A detailed description of the vulnerability.
   - The potential impact (what an attacker could achieve).
   - Step-by-step instructions (or a proof-of-concept script/exploit payload) to reproduce the issue.
   - Any proposed fixes or mitigations (optional).

3. **Confidentiality Commitment**
   We commit to keeping the details of your report confidential during the investigation and remediation phase. We ask that you coordinate with us to allow time to issue a patch before disclosing the vulnerability publicly (Responsible Disclosure).

---

## 🕒 Response and Resolution Timeline

- **Acknowledgement**: We will acknowledge receipt of your vulnerability report within **48 hours** of receiving it.
- **Investigation**: We will investigate the issue and follow up with you regarding our findings, estimated timeline for a patch, or requests for clarification.
- **Resolution**: Once verified, we will work to publish a fix in the repository as quickly as possible.
- **Credit**: If requested, we will happily credit you for the discovery in our release notes or changelog once the patch is published.
